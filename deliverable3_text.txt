Employee Management System (EMS)
CS3009 – Software Engineering

1. Introduction
The Employee Management System (EMS) is a Java-based business system developed using the Scrum methodology to automate employee-related operations.
Sprint 1 established the foundation by implementing authentication, role-based access control, and core employee management.Sprint 2 expanded the system with leave management, attendance tracking, and employee search functionalities, improving workflow automation and operational efficiency.
Sprint 3 marks the final phase, where the system achieved full functionality by integrating payroll processing, document management, and dashboards. These additions transformed the system into a complete enterprise-level solution supporting both operational and strategic decision-making.

2. Project Vision
The refined vision of EMS is to provide a fully integrated, secure, and scalable employee management platform that:
Automates HR processes end-to-end 
Provides real-time workforce insights 
Ensures secure handling of sensitive employee data 
Supports payroll accuracy and compliance 
Enables management decision-making through analytics

6. Structured Specifications (Sprint 3)

6.1 Generate Payroll
Field
Description
Function Name
Generate Payroll
Module
Payroll Management
Description
Calculates and generates salary for employees based on attendance and predefined salary structure
Inputs
Employee ID, Base Salary, Attendance Data, Overtime, Deductions
Processing
1. Retrieve employee salary details
Fetch attendance records 
Calculate total working hours 
Compute gross salary (base + overtime) 
Apply deductions 
Generate final payroll recordOutputs | Salary slip, payroll confirmation messagePreconditions | Employee must exist; attendance data must be availablePostconditions | Payroll record stored in databaseExceptions | Missing attendance data, invalid salary input, database error 

6.2 Upload Employee Document
Field
Description
Function Name
Upload Employee Document
Module
Document Management
Description

Inputs
Employee ID, Document File, Document Type
Processing

Store file securely in system

6.5 Manage Employee Document (Verify/Approve/Remove)
Field
Description
Function Name
Manage Employee Document
Module
Document Management
Description
Allows managers to verify, approve, or remove employee documents
Inputs
Document ID, Action (Approve/Reject/Delete), Comments (optional)
Processing
1. Verify manager authorization
Retrieve document record 
Perform selected action:  • Approve → Change status to “Approved”  • Reject → Change status to “Rejected”  • Delete → Remove document from system 
Save action and update recordOutputs | Action confirmation messagePreconditions | Manager must be logged in; document must existPostconditions | Document status updated or removedExceptions | Unauthorized access, document not found, invalid action
